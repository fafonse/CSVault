Represents components and their static relationships. Useful to include some implementation details (technologies used, etc). Really shows the individual systems and how they connect.

- Annotate the connections with the API's/communication type used
- No bidirectional arrows
	- Caller points to callee
- **NOT a [[Flow Chart|flow chart]]**
- **DOES NOT show data flow**

## Style

### Components
The components are the individual boxes and objects all linked up in the diagram.
- Show their responsibilities/technologies in their box
- Components can have a LOT of code, but should be mostly responsible for single things
